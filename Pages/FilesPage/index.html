Файлы

Данный бот достаточно гибок в использовании. Вы можете полностью настраивать его под себя. Даже составлять конфигурационные файлы XML. Ведь бот работает именно на XML файлах. Если Вы знаете язык C#, тогда Вы можете изучить алгоритмы чтения данных файлов в данном репозитории GitHub.

В боте есть всего один тип файлов - это файл команды.

Если у Вас что-нибудь не получилось, тогда не расстраивайтесь! Встроенный компилятор бота предупредит об ошибках в Вашем файле.

Файл команды

Данный тип файла отвечает за работу кастомной команду.

  • В тег GuildId следует вставлять Id сервера, в который Вы хотите добавить команду.
  • В тег Name следует вставлять имя команды. Имя должно быть в одно слово. Пример: ТестКоманда
  • В тег Actions следует вставлять действия, которые должен выполнять бот. Они должны быть в том порядке, в котором они должны выполняться. Ниже будет рассказано немного больше о командах.
        <?xml version="1.0"?>
        <SerializableCommand xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance" xmlns:xsd="http://www.w3.org/2001/XMLSchema">
            <GuildId>GUILD_ID</GuildId>
            <Name>Kick</Name>
            <Actions>
                <ValueTupleOfCommandActionTypeObject>
                    <Item1>Kick</Item1>
                    <Item2 xsi:type="SerializableKick">
                        <DataFromBuffer>false</DataFromBuffer>
                    </Item2>
                </ValueTupleOfCommandActionTypeObject>
            </Actions>
        </SerializableCommand>

Про команды

Типы команд

В данном разделе мы поговорим про типы команд. В моем боте есть два варианта типов:
  • Serializable
  • Regular
Посмотрим на пример выше. Обратим внимание на данный фрагмент: 
        <Item1>Kick</Item1>
        <Item2 xsi:type="SerializableKick">
Item1 "говорит" боту, что он должен кикнуть какого-то человека (пока что неизвестно каким способом). Item2 "говорит" боту данные нужные для выполнения самой команды. Здесь параметр DataFromBuffer указывает источник информации или же буфер бота. Что такое буфер бота? Приведу в пример действие Interactive. При ее выполнении указываем боту ждать следуещее сообщение от пользователя. Если пользователь отвечает, мы записываем текст сообщения в буфер. И при действии (например, Message) мы используем данные из буфера.

Serializable типы

Их существует всего три.
  1. SerializableKick
  2. SerializableBan
  3. SerializableMessage
По названию, я думаю, можно догадаться их назначение. У всех (кроме SerializableMessage, у него еще есть параметр Message. Он необязателен к заполнению, если значение DataFromBuffer - true) есть буфер.

Regular типы

Их уже четыре.
  1. Ban
  2. Kick
  3. Interactive
  4. Message
Прокомментирую только Interactive. Это действие позволяет ждать следующего сообщения от пользователя.
